こちらが、これまでの開発内容を「目的」「構成」「進捗」「成果物」の4つの観点で整理したまとめです：

---

## 🎯 目的（Why）

音楽フェスティバルの出演アーティスト情報を効率的に管理・閲覧・予習できるWebアプリケーションを構築することで、  
ユーザー（一般参加者）と管理者（運営スタッフ）の双方が、イベントの全体像とタイムテーブルを把握しやすくすることを目指しています。

主な目的：

- 出演アーティスト・イベント情報の一元管理
- Spotify連携による予習体験の提供
- タイムテーブルの登録・表示・編集による運営支援

---

## 🧱 構成（What）

### 技術スタック

| 項目         | 使用技術                          |
|--------------|-----------------------------------|
| フレームワーク | Django                            |
| 言語         | Python, HTML, CSS, JavaScript     |
| データベース | SQLite（Django ORM）              |
| 外部API      | Spotify Web API                    |
| 環境管理     | `.env` + `python-dotenv`          |

### 機能モジュール

- アーティスト管理（一覧・詳細・検索・Spotify連携）
- イベント管理（日程・会場・出演者）
- プレイリスト作成（Spotify代表曲取得）
- タイムテーブル管理（登録・表示・編集）

## 🚧 進捗（How）

### 改善済みポイント

- `Performance` モデルに `unique_together` を追加し、出演者の重複登録を防止
- `selected_artist_ids` のPOST引き継ぎ漏れを修正
- 表示画面で `event_day` による絞り込みを導入し、他日程の混入を防止
